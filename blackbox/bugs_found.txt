TITLE:
Multiple default addresses remain after adding a new default address.

ENDPOINT TESTED:
Method: POST
URL: http://127.0.0.1:8080/api/v1/addresses

Headers:
X-Roll-Number: 2024101087

Body:
{
  "label": "OTHER",
  "street": "98765 Default Switch Road",
  "city": "Pune",
  "pincode": "654321",
  "is_default": true
}

Precondition Step Executed: GET /addresses and confirm one existing default address.

HTTP Status: 200

ACTUAL RESULT:
HTTP Status: 200
New default address is created, but previous default is not cleared.
Result: multiple addresses have "is_default": true.

ACTUAL JSON :
{
  "message": "Address added successfully",
  "address": {
    "address_id": 1001,
    "label": "OTHER",
    "street": "98765 Default Switch Road",
    "city": "Pune",
    "pincode": "654321",
    "is_default": true
  }
